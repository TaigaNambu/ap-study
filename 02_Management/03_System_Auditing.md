# 3. システム監査

## 3.1 監査の基礎と手順

### 1. システム監査
情報システムのリスクに対するコントロール（統制）が適切に整備・運用されているかを、**独立した客観的な立場**の監査人が検証・評価し、助言を行う活動。
* **目的:** 情報システムの信頼性、安全性、効率性の向上。

### 2. 監査人の独立性
監査人は、監査対象部門から独立していなければならない。
* **外観上の独立性:** 利害関係がないこと。身内や以前所属していた部署の監査は避ける。
* **精神的な独立性:** 公正不偏の態度を保つこと。

### 3. 監査の手順
1. **監査計画の策定:** 中長期計画に基づき、個別監査の目的、範囲、スケジュールを決定する。
2. **予備調査:** アンケートや資料収集により、現状を概略的に把握する。
3. **本調査:** 証拠資料の収集、ヒアリング、現地調査を行い、監査証拠を入手する。
4. **評価・結論:** 集めた証拠に基づき、指摘事項をまとめる。
5. **監査報告:** 監査報告書を作成し、依頼者（経営者など）に報告する。
6. **フォローアップ:** 監査対象部門による改善実施状況を確認する（改善指導自体は行わない場合が多い）。

## 3.2 内部統制とコントロール

### 4. 内部統制 (Internal Control)
業務の有効性・効率性、財務報告の信頼性、法令遵守、資産の保全を確保するために、組織内に組み込まれた仕組み。

### 5. 職務分掌 (Segregation of Duties)
不正や誤謬を防ぐために、担当者の役割や権限を適切に分離すること。
* **例:** 「現金を扱う人」と「帳簿をつける人」を別にする。「システムの開発者」と「本番環境への移行担当者」を分ける。

### 6. ITガバナンス
企業が競争優位性を構築するために、IT戦略の策定・実行をコントロールする組織的な能力。経営陣が責任を持つ。
