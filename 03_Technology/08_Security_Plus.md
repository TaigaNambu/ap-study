# 8. 高度セキュリティ・ネットワーク

## 8.1 安全な通信プロトコル

### 1. VPNとセキュア通信
* **IPsec (Security Architecture for IP):** ネットワーク層（IPパケット単位）で暗号化を行うプロトコル。VPN（拠点間接続）構築によく利用される。
* **SSL / TLS:** トランスポート層以上で暗号化を行う。Webブラウザ(HTTPS)などで標準的に利用される。
* **S/MIME:** 電子メールの暗号化と電子署名を行う規格。

### 2. 認証プロトコルとID管理
* **OAuth 2.0:** 認可（権限）の委譲を行うフレームワーク。「Googleアカウントでログイン」などで利用。
* **OpenID Connect:** OAuth 2.0をベースに、認証（本人確認）を行うための規格。
* **FIDO (Fast IDentity Online):** パスワードを使わず、生体認証などを利用して安全に認証する規格。
* **多要素認証 (MFA):** 知識（パスワード）、所持（スマホ）、生体（指紋）のうち2つ以上を組み合わせる認証。

## 8.2 組織的なセキュリティ対策

### 3. セキュリティ組織と対応
* **CSIRT (Computer Security Incident Response Team):** セキュリティ事故が発生した際に、対応を行う専門チーム。
* **SOC (Security Operation Center):** ネットワークやデバイスを24時間体制で監視・分析する組織。
* **SIEM:** 様々な機器のログを一元管理・分析し、脅威を検知するシステム。

### 4. 攻撃手法の深化
* **ランサムウェア:** データを暗号化して「人質」にとり、復旧と引き換えに身代金を要求するマルウェア。
* **ゼロデイ攻撃:** ソフトウェアの脆弱性が発見されてから、修正パッチが提供されるまでの隙を突く攻撃。
* **APT攻撃 (Advanced Persistent Threat):** 特定の組織を執拗に狙い、長期間潜伏して情報を盗み出す攻撃。
