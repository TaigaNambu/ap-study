# 5. セキュリティ

## 5.1 情報セキュリティの基礎

### 1. 情報セキュリティの3要素 (CIA)
* **機密性 (Confidentiality):** 許可された人だけがアクセスできること。
* **完全性 (Integrity):** 情報が改ざんされておらず、正確であること。
* **可用性 (Availability):** 必要な時にいつでも使えること。

### 2. 暗号技術
* **共通鍵暗号:** 暗号化と復号に「同じ鍵」を使う。高速だが、鍵の配送に課題がある。（例: AES）
* **公開鍵暗号:** 「公開鍵」で暗号化し、「秘密鍵」で復号する。鍵配送問題は解決するが、計算が遅い。（例: RSA）
* **ハイブリッド暗号:** 両者を組み合わせ、共通鍵を公開鍵暗号で送る方式。SSL/TLSなどで利用。

### 3. 認証技術
* **電子署名:** 送信者の「秘密鍵」で暗号化（署名）し、受信者が「公開鍵」で復号（検証）する。「送信者の正当性」と「改ざんされていないこと」を証明する。
* **PKI (公開鍵基盤):** CA（認証局）が発行するディジタル証明書を用いて、公開鍵の正当性を保証する仕組み。

## 5.2 脅威と対策

### 4. サイバー攻撃手法
* **SQLインジェクション:** 入力フォームに不正なSQLを注入し、DBを不正操作する。
* **XSS (クロスサイトスクリプティング):** 悪意あるスクリプトを埋め込み、閲覧者のブラウザで実行させる。クッキー盗難などに繋がる。
* **DoS / DDoS攻撃:** 大量のデータを送りつけ、サービスを停止させる。
* **フィッシング:** 偽サイトに誘導し、個人情報を盗む。
* **標的型攻撃:** 特定の組織を狙って、業務関連を装ったウイルスメールなどを送る。

### 5. セキュリティ対策
* **ファイアウォール:** パケットフィルタリングにより、不正な通信を遮断する。
* **WAF (Web Application Firewall):** Webアプリへの攻撃（SQLインジェクションなど）を検知・防御する。
* **DMZ (非武装地帯):** 外部（インターネット）と内部（LAN）の間に設ける緩衝地帯。Webサーバなどを配置する。
