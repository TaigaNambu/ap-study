# 3. ネットワーク

## 3.1 ネットワークアーキテクチャ

### 1. OSI参照モデル (7階層)
1. **物理層:** ケーブル、電気信号 (リピータハブ)
2. **データリンク層:** 隣接ノード間の通信、MACアドレス (L2スイッチ)
3. **ネットワーク層:** 経路制御、IPアドレス (ルータ、L3スイッチ)
4. **トランスポート層:** 通信品質の確保、ポート番号 (L4スイッチ)
5. **セッション層:** 通信の開始・終了手順
6. **プレゼンテーション層:** データの表現形式（文字コード、暗号化）
7. **アプリケーション層:** 利用者へのサービス提供 (HTTP, SMTP)

### 2. TCP/IPプロトコル群
* **IP (Internet Protocol):** パケットの宛先への配送（コネクションレス型）。
* **TCP (Transmission Control Protocol):** 信頼性の高い通信（コネクション型）。再送制御や順序制御を行う。3ウェイハンドシェイクで接続確立。
* **UDP (User Datagram Protocol):** 信頼性より速度重視（コネクションレス型）。動画配信やDNSで使用。

## 3.2 ネットワーク管理と応用

### 3. IPアドレスとサブネット
* **IPv4:** 32ビットのアドレス。
* **サブネットマスク:** IPアドレスのうち、どこまでが「ネットワーク部」で、どこからが「ホスト部」かを示すビット列。
* **CIDR:** クラスを使わず、プレフィックス長（例: /24）でネットワーク部を表す方式。
* **プライベートIPアドレス:** 組織内でのみ有効なアドレス。インターネットには直接接続できない。

### 4. ネットワークサービス
* **DNS (Domain Name System):** ドメイン名とIPアドレスを対応付けるシステム。
* **DHCP (Dynamic Host Configuration Protocol):** IPアドレスなどのネットワーク設定を自動的に割り当てるプロトコル。
* **NAT / NAPT (IPマスカレード):** プライベートIPアドレスとグローバルIPアドレスを変換する技術。NAPTはポート番号も使って1つのグローバルIPで複数台接続可能にする。

### 5. メールプロトコル
* **SMTP:** メールの送信・転送。
* **POP3:** メールの受信（サーバからダウンロードして削除）。
* **IMAP4:** メールの受信（サーバ上でメールを管理）。
